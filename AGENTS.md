## Project Overview

RECORD is a full-stack web application with three sibling projects:
- **`api/`** — OpenAPI 3.0 contract (source of truth for both sides)
- **`backend/`** — Kotlin + Spring Boot 3.5 REST API
- **`frontend/`** — Angular 21 SPA

## Commands

### Backend (run from `backend/`)

```bash
docker compose up -d          # Start Postgres (required before bootRun)
./gradlew bootRun             # Run the server (http://localhost:8080)
./gradlew test                # Run tests (uses H2 in-memory, no Postgres needed)
./gradlew build               # Build + regenerate OpenAPI stubs
./gradlew openApiGenerate     # Regenerate OpenAPI stubs only
```

### Frontend (run from `frontend/`)

```bash
pnpm install                  # Install dependencies
pnpm start                    # Dev server at http://localhost:4200
pnpm test                     # Run tests (Vitest)
pnpm build                    # Production build
pnpm generate:api             # Regenerate Angular API client from ../api/openapi-rest.yaml
```

## OpenAPI Code Generation

The `api/` directory is the single source of truth. **Never manually edit generated files.**

- Backend stubs: `backend/build/generated/openapi/` — regenerated by every `./gradlew build`
- Frontend client: `frontend/src/app/core/api/` — regenerated by `pnpm generate:api` (git-ignored)

When adding or changing an endpoint: update `api/openapi-rest.yaml` (and its referenced YAML files), then regenerate both sides.

## Backend Architecture

Spring Modulith + Hexagonal Architecture. Each domain is a bounded module under `es.uib.record.backend/`:

```
{domain}/
├── application/usecase/   # Use cases (application services)
├── domain/                # Aggregates, repository ports, domain exceptions
├── infrastructure/
│   ├── adapter/           # Facade implementations (cross-module)
│   ├── controller/        # Implements generated OpenAPI interface
│   ├── mapper/            # Domain ↔ Entity / Domain ↔ DTO mappers
│   └── persistence/       # JPA entities, Spring Data repos, repository adapters
└── open/                  # Public facade interfaces for cross-module access
```

Current modules: `auth`, `users`, `groups`, `security`, `shared`.

**Key patterns:**
- Controllers implement the generated interface (e.g., `AuthController : AuthApi`) — never drift from the contract.
- Cross-module dependencies go through `open/` facade interfaces only (e.g., `UserFacade`, `AuthFacade`). Never import across module internals.
- Domain exceptions extend `DomainException(message, code, type, params)` with an `ErrorType` enum value. `GlobalExceptionHandler` maps `ErrorType` to HTTP status codes.
- Flyway migrations live in `backend/src/main/resources/db/migration/V{N}__description.sql`.

## Frontend Architecture

Feature-based Angular 21 under `frontend/src/app/`:

```
core/
  api/          # Generated OpenAPI client (git-ignored)
  guards/       # authGuard, guestGuard
  interceptors/ # authInterceptor (Bearer injection + auto-refresh on 401)
  services/     # UserState (Angular signals)
features/       # auth, dashboard, settings
layouts/        # main-layout (authenticated shell)
```

**Key patterns:**
- TypeScript path aliases: `@core/*`, `@features/*`, `@layouts/*`
- Component selector prefix: `record-` (e.g., `record-login`)
- Auth tokens stored in `localStorage` (`access_token`, `refresh_token`); interceptor handles refresh automatically
- i18n via ngx-translate; translation files at `frontend/public/i18n/{lang}.json` (available languages are `en`, `es`, `cat`)
- UI: PrimeNG 21 + PrimeFlex; dark mode via `.dark` class on `<html>`

```
JWT_EXPIRATION=
JWT_REFRESH_EXPIRATION=
DB_NAME=
DB_USER=
DB_PASSWORD=
CLOUDINARY_CLOUD_NAME=
CLOUDINARY_API_KEY=
CLOUDINARY_API_SECRET=
CLOUDINARY_USER_AVATAR_FOLDER=
CLOUDINARY_USER_TRANSFORMATION=
```

Swagger UI is available at `http://localhost:8080/swagger-ui.html` when the server is running.
